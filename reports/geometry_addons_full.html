1. O problema técnico

Você viu nos experimentos anteriores algo que te frustra: o material parece lindo no Blender, exporta o GLB, e no browser fica chapado, plástico, "fraco". Não é coincidência. Há uma assimetria entre o que o Blender renderiza e o que o glTF 2.0 sabe carregar.

A verdade incômoda O exportador glTF do Blender descarta silenciosamente quase tudo que não termina num Principled BSDF apontando para texturas raster. Node procedural (Voronoi, Noise, Object Coord), displacement de shader, custom shader nodes, Geometry Nodes, vertex colors em alguns casos — nada disso sobrevive nativamente.

O que sobrevive ao export GLB:

  • Geometria de mesh (vértices, faces, normals, UVs) — desde que aplicada como mesh estático
  • Principled BSDF com base color, roughness, metallic, normal map, occlusion, emission — desde que via texturas raster
  • Skin / armature / animations (como vimos no experimento de rigging)
  • Morph targets / shape keys
  • Vertex colors (parcialmente)

O que não sobrevive:

  • Shader procedural (qualquer node tree fora do Principled puro)
  • Displacement de shader (não move vértices, é só ilusão de paralaxe)
  • Geometry Nodes não aplicados (precisa "apply" antes do export)
  • Particle systems (cabelo, fur, fumaça) — sem virar mesh primeiro
  • Cloth simulation não baked
  • Modifiers procedurais (Subsurf, Array, Boolean) — precisa apply ou são ignorados

2. Critério de avaliação

Cada addon foi classificado em 3 tiers:

Tier 1 — Drop-in para GLB

Gera mesh real direto. Workflow: rodar → apply modifiers → export. Sem pegadinhas.

Compra prioritária se cobrir a necessidade.

Tier 2 — GLB com bake intermediário

Geo Nodes / sim que precisa apply + bake de shader pra texturas raster antes do export. Funciona, com workflow extra.

Vale se feature for única.

Tier 3 — Pipeline tools

Não geram material, mas otimizam o pipeline: retopo, decimate, bake automation, GLB optimization.

Compra estratégica de produtividade.

9. Pipeline canônica Blender → GLB com fidelidade

Independente do addon escolhido, esta sequência é o que separa "fica feio no GLB" de "fica fiel".

1
Modelar / importar a base
Pode ser primitivo, mesh comprado, scan, modelo gerado por IA, Suzanne. O ponto de partida.
2
Aplicar addon de geometria
Easy Cardboard / Trowel / Clay Doh / KnitGen / OCD / o que for. Roda em modo "vivo" como modifier ou geometry node setup.
3
Iterar e ajustar parâmetros
Aproveitar que ainda é procedural pra variar layers, wear, padding. Gerar várias versões do mesmo objeto.
4
Apply all modifiers
Critério não-negociável. Ctrl+A → Apply All Modifiers ou via Python bpy.ops.object.convert(target="MESH"). Geometry Nodes / modifiers procedurais viram mesh estático aqui.
5
Cleanup do mesh
Mesh Cleaner 2 (grátis) ou Instant Clean: remove doubles, fix normals, fill holes, weld vertices. Sem isso, normals quebradas no GLB.
6
Decimate inteligente / retopo
Geometry Nodes facilmente gera 500k-2M tris. Pra web, decimar pra 50-150k tris preservando silhueta. Decimate modifier nativo (Planar) ou Quad Remesher / Instant Meshes (preserva topologia melhor).
7
UV unwrap
Smart UV Project ou unwrap manual. Sem UV bom, o bake do material fica esticado.
8
Bake PBR
SimpleBake ou Principled Baker. Gera baseColor.png + roughness.png + metallic.png + normal.png a partir do shader procedural. Resoluções recomendadas: 2K pra web, 4K se hero asset.
9
Reconstruir material como Principled BSDF puro
Trocar o shader procedural por Principled BSDF apontando pras texturas baked. Esse é o ÚNICO material que glTF exporta com fidelidade.
10
Export GLB
Settings: export_format="GLB", export_apply=True, export_skins=True (se rig), export_animations=True (se anim), export_force_sampling=True (anim).
11
(Opcional) Pós-processamento glTF-Transform
GLB Optimizer ou CLI: gltf-transform optimize. KTX2 compression (texturas), draco compression (mesh), dedup, prune. Pode reduzir GLB 30-60%.

11. Armadilhas comuns e como evitar

Armadilha Sintoma Solução
Shader procedural não baked GLB branco translúcido Bake PBR antes do export. Use SimpleBake/Principled Baker.
Geometry Nodes não aplicado GLB sem a geometria gerada Apply All Modifiers antes do export.
Particle hair/fur GLB sem cabelo Generate Hair as Mesh (free) converte pra mesh strands.
Displacement de shader Mesh continua liso no GLB DisplaceIt converte para deslocamento de mesh real.
Cloth sim não baked GLB com pose inicial, não a drapped Bake to keyframe / Apply modifier no frame final.
Poly count alto GLB 50MB+, lento no browser Decimate planar + Mesh Cleaner antes do bake. Gltf-transform draco compress pós-export.
Normal map invertido Sombras erradas no Three.js Bake Normal com space="TANGENT", flip_y se necessário. Validar no Three.js debug.
Material parallax (DECALmachine) Detalhe sumido no GLB Bake o shader parallax → normal map antes do export.
Multi-material com UV diferentes Texturas misturadas Texture atlas via OneAtlas ou SimpleBake atlas mode.
Subsurface scattering Material plástico no GLB SSS não exporta direto. Compensar com Sheen + base color mais saturada no Principled BSDF.
